One simple hand-off, every kind of test

Every model is treated the same way: you hand it an input (a set of images, a sentence, a sound, a video, a moment in a game, even a switched-off part of the model) and it hands back a response (where the brain lights up, a choice, an action, or a written answer). The kind of input decides what's being tested; the kind of response tells you how it did. That single, shared hand-off is what lets one model run on every test below.

Finding a model's word-selective units

To study reading, we first have to find the model's word-selective units. We do it the way brain scientists do: show the model words and non-words, and keep the neurons that fire much harder for real words.

Gathering them from wherever they live

The usual approach reads from a single layer of the model; here, instead, we collect the word-selective units from across several layers and treat them as one group, because that's where they naturally turn out to be.

Caveats

Switching off a model's word-selective units: giving it "dyslexia"

We switch off the model's word-selective units, re-test its reading, then switch them back on (it restores perfectly). The result reproduces a 2026 study: in a large enough model, switching off the word-selective units pushes its reading below the dyslexia line, while switching off the same number of random neurons barely matters. And the effect only appears once the model is big enough.

Better models, better scores

A test worth running should reward better models and sit near its "knows-nothing" floor for an untrained one. That's the basic sanity check: pick a capability and see whether the scores climb the way they should.

A null baseline, for every test

No result counts here until it clearly beats a deliberately broken version of the same test: the same machinery, with the real signal stripped out. If a score doesn't clear that floor, it's coming from the setup, not the model.
