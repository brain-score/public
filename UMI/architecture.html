The contracts

The full surface. These are the only things a model author touches; everything else (dispatch, caching, compatibility, memory) is handled by the framework.

1 · The UnifiedModel ABC

The minimal interface every model satisfies. No capability-named methods (look_at, digest_text) — capability is derived from the preprocessors present.

class UnifiedModel(ABC):
    def process(self, input_event: InputEvent,
                multi_modality: bool = False) -> OutputEvent: ...
    def start_recording(self, recording_target: str | list[str],
                        time_bins=None, recording_type=None) -> None: ...
    def start_task(self, task_context: TaskContext) -> None: ...
    def reset(self) -> None: ...

    @property
    def supported_modalities(self) -> set[str]:   # derived from preprocessors.keys()
        ...

2 · The BrainScoreModel registration

The concrete class. A registration is one constructor call; the optional slots are what decide which dispatch branches the model can take.

BrainScoreModel(
    identifier: str,
    model,                              # the nn.Module / backbone (None for output-only)
    region_layer_map: dict,             # brain region -> layer path
    preprocessors: dict[str, Callable], # modality -> transform  (THIS defines supported_modalities)
    activations_model=None,             # wrapper(s): PytorchWrapper / Text / VLMVision / Video / Audio
    generation_fn=None,                 # behavioral: (row, instruction, label_set) -> str
    action_fn=None,                     # embodied:   (EnvironmentStep) -> EnvironmentResponse
    state_change_fn=None,               # lesion:     (StateChange) -> (PerturbationApplied, cleanup)
    behavioral_readout_layer=None,      # fit a logistic head on this layer
    region_modality_map=None,           # multimodal: region -> which tower extracts it
    backbone_id=None,                   # share a feature cache across registrations
)

3 · Input events — the InputEvent union

EventCarriesDispatches to
StimulusSetcolumns for one modality (image / text / audio / video)preprocessor → activations_model (or behavioral path)
MultimodalStimulusSetcolumns for several modalities + per-modality sampling ratesone or every supported tower
StateChangeSelection (which units) + Perturbation (zero/scale/replace), or a reset handlestate_change_fn
EnvironmentStepcameras + proprioception + instruction (DROID-shaped), or a generic observationaction_fn
Messagean inter-agent message (multi-agent rollout)action_fn (responder)

These differ along two axes, not one. Content: they all present perceptual/linguistic input — an EnvironmentStep doesn't sit beside StimulusSet, its observation is a (Multimodal)StimulusSet, and a Message is wrapped to EnvironmentStep(observation=message) at dispatch. Interaction: StimulusSet is a batched, open-loop, stateless set (N presentations → one assembly), while EnvironmentStep / Message are single-step, closed-loop, stateful (the output feeds the next input → a response). You can't batch a closed loop — step t+1 doesn't exist until the model acts at t — so the split is that interaction axis, not the content (which is already shared).

5b · Time — the time_bin axis + the temporal kit

Naturalistic benchmarks need when, not just what. Every time-aware wrapper emits a time_bin dim with canonical coords (time_bin_start_ms / time_bin_end_ms / time_bin_center_ms), so a benchmark aligns any wrapper's output uniformly — no per-wrapper special case. core/brainscore_core/temporal.py is the shared kit:

HelperDoes
temporal_binaggregate per-frame/token features into requested time windows
hrf_convolveBOLD-aware: convolve model features with the hemodynamic response before fitting fMRI
contiguous_block_cvblock cross-validation for autocorrelated time series
window_plantile a clip longer than a model's native window into block/causal windows
add_time_bin_axispromote a still-image wrapper's output to a single time_bin

A clip longer than a video model's native temporal window does not silently downsample: VideoWrapper(context_window_ms=…) uses window_plan to chunk it, forwards each window, and stitches the per-window time-resolved features into one clip-time sequence with absolute timestamps (max_clip_ms gives a fail-fast). Model→brain time alignment lives in the benchmark (temporal_bin + hrf_convolve), not a commitment layer — the same reason regions map to layers freely rather than through a fixed hierarchy.

6 · The dispatch order — inside process()

def process(input_event, multi_modality=False):
    if isinstance(input_event, StateChange):     return state_change_fn(...)      # PerturbationApplied
    if isinstance(input_event, EnvironmentStep): return action_fn(...)            # EnvironmentResponse
    if isinstance(input_event, Message):         return action_fn(...)            # Message | EnvironmentResponse

    # --- perceptual (StimulusSet) ---
    if generation_viable_for_task():             return _generate_predictions()   # BehavioralAssembly
    if readout_viable_for_task():                return _predict_probabilities()  # BehavioralAssembly

    detected = _detect_modalities(stimuli)        # by column name
    layers   = recording_layers or [recording_layer]
    if multi_modality and len(detected) > 1:      # fan out, concat w/ modality coord
        ...
    else:                                         # MODALITY_PRIORITY = (vision, text, audio, video)
        modality = pick_by_priority(detected)
    return activations_model(preprocessors[modality](stimuli), layers)            # NeuroidAssembly

7 · The compatibility contract

Run as a pre-flight check (then a memory probe) before any benchmark, so mismatches fail in milliseconds instead of mid-extraction:

benchmark.required ⊆ model.available     # the model can take what the benchmark gives
model.required    ⊆ benchmark.required   # the benchmark provides what the model demands

A benchmark declares exactly one input format via required_modalities; a model declares required ⊆ available. GPT-2's required={text} is what makes it fail fast on a V4 image benchmark. check_memory() then runs one stimulus through process(), measures peak memory (CUDA / MPS / CPU), and extrapolates to the full benchmark.

Extending it — five inherited seams + one new

The interface is a backbone: a small, stable core you extend along well-defined seams. Model / Benchmark / Metric / Data / Stimulus-set are Brain-Score's existing plugin registries, reused unchanged; Capability is the only seam the unified interface adds. We ship the seams + a few reference integrations; you bring the breadth. None of this touches the core.

One mechanism: registries

Everything is a factory registered under a string identifier. A plugin is a subpackage whose __init__.py adds a factory to the matching registry; you then load by id.

import brainscore
data      = brainscore.load_dataset('your-data')          # vision/language registry
stimuli   = brainscore.load_stimulus_set('your-stimuli')  # reused across benchmarks
metric    = brainscore.load_metric('your-metric')         # the metric seam, now first-class
benchmark = brainscore.load_benchmark('your-benchmark')
score     = brainscore.score('your-model', 'your-benchmark')

load_* checks the unified registry first, then falls back to the vision / language registries, so legacy plugins keep working. Five registries — data, stimulus_set, metric, benchmark, model — are Brain-Score's; the unified interface adds the Capability slots.

The Metric seam is where the field is moving — beyond linear predictivity toward representational, topographic, behavioral, and causal alignment. Each distinct axis is a distinct metric (e.g. load_metric('topographic-alignment') scores a model's spatial unit layout against cortical topography, which predictivity is blind to — it is permutation-invariant over units). Filling an existing Capability slot needs no core change at all; only a genuinely new input/output type touches the core dispatch.

The topographic axis, validated on real fMRI. Scored on an NSD fsaverage-surface target through this exact seam:

modeltopographic alignment (signal vs. shuffle null)
CLIP ViT-B/32 (non-topographic)−0.155 — sits below the null (correct negative control)
TDANN (topographic ResNet-18)+0.838 — its unit layout matches the cortical sheet

Predictivity cannot tell these apart; the topographic metric + a position-shuffle null can. There are two topographic axes: this one (units correlated across stimuli, which TDANN is organized on) and a selectivity axis (units sharing category preference, which a model like Topo-Omni is organized on) — distinct metrics, because the metric must match the axis the model was smoothed on.

What the backbone deliberately leaves to you: discovery / analysis pipelines (they consume process() output — ship them as scripts, not core primitives), model-internal descriptors that don't compare to the brain, model training, and catalog breadth. Full guide: EXTENDING.md + templates/ in the repo.
